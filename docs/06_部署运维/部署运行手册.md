# 部署运行手册

## 1. 适用范围

本手册用于当前 `go_chat` 项目的本地、测试机和云服务器部署。

当前推荐部署方式：Docker Compose。

## 2. 启动前检查

- Docker / Docker Compose 已安装
- 目标机器端口未被占用：`8080`、`3306`、`6379`
- 项目目录具备写入权限（上传目录、数据卷）
- 生产环境已替换默认数据库与 JWT 凭据

## 3. 关键配置

- 推荐先复制 Docker 环境模板：`cp .env.docker.example .env.docker`
- `GO_CHAT_APP_PORT`：应用对外端口，默认 `8080`
- `MYSQL_ROOT_PASSWORD`
- `MYSQL_DATABASE`
- `MYSQL_USER`
- `MYSQL_PASSWORD`
- `MYSQL_DSN`
- `REDIS_ADDR`
- `REDIS_PASSWORD`
- `REDIS_DB`
- `JWT_SECRET`
- `JWT_EXPIRE_HOURS`
- `UPLOAD_DIR`
- `UPLOAD_MAX_IMAGE_MB`

说明：历史测试报告中出现的 `18080` 仅为阶段验证端口，不作为当前默认部署参数。
说明：若仓库为公共仓库，不要把生产密钥直接写回 `docker-compose.yml`，应保存在服务器本地 `.env` 或 `docker-compose.override.yml`。

## 4. 启动命令

### 默认端口启动

```bash
docker compose --env-file .env.docker up -d --build
```

### 自定义端口启动

```bash
GO_CHAT_APP_PORT=18080 docker compose --env-file .env.docker up -d --build
```

## 5. 启动后验证

```bash
docker compose --env-file .env.docker ps
docker compose --env-file .env.docker logs -f app
```

推荐最小验证项：

1. 注册接口可用
2. 登录接口可用
3. WebSocket 可建连
4. MySQL / Redis 容器健康检查通过

## 6. 停止与清理

```bash
docker compose --env-file .env.docker down
docker compose --env-file .env.docker down -v
```

注意：`docker compose down -v` 会删除 MySQL / Redis 数据卷。

## 7. 回滚建议

- 代码回滚：回退到上一稳定版本镜像或提交
- 数据回滚：恢复 MySQL 备份，必要时恢复 Redis 持久化数据
- 配置回滚：保留上一版环境变量与 compose 配置副本
